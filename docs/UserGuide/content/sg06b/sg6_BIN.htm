BIN

Syntax:

BIN( data:Range or Array, bins:Range or Array )

The bins range of cells (or array) defines a set of bins into which the values from the data range of cells (or array) is counted.

For example, if bins is {20,40,60,80}, the returned array gives the number of elements of data that have:

  •         value ≤20
  • 20< value ≤40
  • 40< value ≤60
  • 60< value ≤80
  • 80< value

The returned array will therefore always have one more element than the bins range of cells (or array). Neither data nor bins need be sorted.

Unlike the FREQUENCY() spreadsheet function, BIN() can compare non-numeric values, e.g. bins {"A","B","C","D","E"} is valid.

One common use of this function is to organise data for making a histogram.

Example:

With data in A1:A100, and bin categories in B1:B5, the following formula would provide the basis for a bar chart: 

SET_VALUE(C1:C6, BIN(A1:A100, B1:B5))

You would then make a bar chart from the range B1:C6, using column B as labels.

Revisions:

Starting with 2.00, all the supplied arbitrary dimensioned data range of cells (or array) is binned. Prior to that only the first column of this data would have been binned.
