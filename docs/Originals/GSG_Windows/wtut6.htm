Combining text and numbers

This chapter shows you further ways in which you can use Fireworkz to simplify tasks which would be much more complicated if you were using separate word processor and spreadsheet programs.

In this chapter you will:

Work on a report which contains a mixture of text, tables containing live data, and charts.

Checking the text

Open the document Report.fwk from the Tutorial directory. You’ll see that it already contains a couple of sections of text, some headings and a couple of tables. We will be adding to this in the course of this chapter, as well as adding more formatting to the contents of the document.

Scroll through the document to familiarise yourself with its contents. You’ll see that there is a faint grid around all the cells and paragraphs.

Click on the Style button to see the list of styles which has already been set up in this document. There are styles for two levels of heading, body text (the base style), table text, table heading, and table numbers.

This document was based on a sheet template. It started off as a plain worksheet, and the text was then added to it. Because of this, the document’s tables don’t yet have borders set up around them other than the faint grid.
Adding a border to a table

The next step is to add a grid to make the table easier to read when the document is printed out.

Because of the way Fireworkz draws grids, it’s always a good idea to leave a row/paragraph between the top of a table and the surrounding worksheet. You’ll see that there is such a pair of buffer rows in the document already.

You can shrink this down to around 0.3cm by pointing at the border between these rows and the row beneath, and dragging the row resize pointer upwards. You can check the measurement of the row either by looking at the ruler or by reading the measurement in the status line.

Now select the whole table by dragging from b6 to f10, and choose Box from the Style menu. The Box dialogue will appear.

Click on All, and click OK.

The box will be drawn with standard thickness black lines.

Referring to numbers in text

The text in this document mostly describes the contents of the tables. As such, it is helpful to refer directly to the table contents. Because the data in the tables is preliminary, and is likely to be updated, one option is to use names to refer to the data. If the data is updated in the table, the copy elsewhere in the document is updated too.

There is already an example of this set up in the document. In the paragraph beneath the first table, a line of text leads to a cell containing the contents of the name Redbang.

Setting up a name

Your first task when you want to use a name is to define it.

Click on the Function selector button.

Choose Names from the menu which appears.

A dialogue box listing names which have already been defined will appear. Click the New button.

A further dialogue will appear in which you should define the name. You need to enter a name for the name, as well as its definition. Enter Worst for the name and the cell reference f8 for the definition.

Click OK. Your new name will be added to the list.

Referring to a name in a number cell

Using names is simple. You simply pick a name from the list and enter it, and the data which the name refers to.

Click on cell c13.

Click on the function selector and choose Names from the menu again.

Select Worst and click on--Insert. The name will be added to the formula line.

Click the Tick button or press Return to enter the number in the cell.

Using names in this way is the same as using a cell reference, but is easier to manage. For example, you can use names created in one document in any other document which is open. It’s much easier to work with a name in a second document than a cell reference, because the name will always remind you of its origins.

To see a name working in this way, click on the Fireworkz icon and open a new sheet based document. Click the Function selector button and choose Names from the menu which appears. You’ll see your names appear in the list, along with their full path names. You’re not going to work with this second document (it was used purely as an example), so close it without saving it.

Referring to a name in a text cell

The second method of using names enables you to use the data contained in a name within text. You can see this in action in row 4 of the document, a long text paragraph. The last sentence has the figure entered as a name.

Move to row 15 of the document. This requires the name you defined to be entered in the text.

Place the caret before the comma between at and must in the last line.

Choose Name from the Inserts submenu from the Extra menu.

Choose Worst from the dialogue box which appears.

Click Insert. The value of the name, in this case 5.45, will be added to the text.

Updating your data

The benefit of using names is that the references are automatically updated when the original data changes. To see this in action:

Click on cell e9 in the table.

Click in the formula line and select the numbers which appear.

Enter 8 as a replacement number.

Click the Tick button or press Return. The table cell will be updated and so will the references to it in both the text paragraph above, row 4, and the cell below, row 12.
Drawing a chart

While tables are a good way of presenting numerical information, readers have to examine them carefully to interpret the data they contain. Charts, on the other hand, provide a visual representation of data which is obvious to most readers.

You can draw a chart from any data held in a table. Numerical data used as the basis for a chart must be held in number cells. The numbers in the tables in this document are all held in number cells, so that isn’t a problem.

Scroll down the document to the end. You’ll see that the last paragraph of text discusses the heights achieved by the different fireworks mentioned in the table. A chart is a great way to show this information, a pictogram even better.

Scroll back up to the table and select cells b7 to c10.

Click on the Chart button. A menu will appear.

Choose Bar. The bar charts gallery will appear:

Click on the third button down in the second column, the one with different sized penguins. This will produce a pictogram chart with the bars made up from a single image.

Check the 3D option is off, then click OK.

Customising the chart

Your chart will be drawn with the default pictogram pictures (penguins) included. What you really want is an appropriate picture to add real sparkle to your report.

Locate the sprite Firepic.ff9 from the Tutorial directory.

Go back to the report document. Double-click inside the chart area to make the chart active, then click the columns to select them. Notice the message in the status line which tells you the series which are selected.

Drag the sprite icon on to the chart.

The chart will redraw with the firework picture used to make up the bars.

Positioning the chart

The final step is to position the chart at the end of the document.

Click once in the chart to select it as a picture.

Drag the chart down to the end of the document. It will appear on the second page.

When the chart is beneath the text describing its contents, it is in the right place.
Finishing the report

The final stage is to save your document and print it.

Saving

To save:

Click the back icon of the window border so that you can see a suitable directory window, or click on the icon of the disc on which you want to save the file, so that the required directory window is visible.

Click on the Save button, or choose Save from the File menu.

The Save dialogue box will appear. Type in a name for the file , choose the drive and directory where you want to save it, and click OK.

Printing

To print the document:

Click on the Print button or choose Print from the File menu. The Print dialogue box appears.

Click OK. The document will be printed.
